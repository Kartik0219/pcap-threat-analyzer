# Sample captures for manual demos

The automated test suite uses small synthetic captures generated by
`tests/fixtures/generate.py`, so it doesn't depend on any external
downloads. To see the tool work against **real** malicious traffic for
a demo or write-up, grab captures from these public, research-oriented
sources (no authorization needed — they're published specifically for
this purpose):

- **Malware-Traffic-Analysis.net** — https://www.malware-traffic-analysis.net/
  Hundreds of labeled captures with write-ups (look at the "Traffic
  Analysis Exercises" section for scenarios with known answers you can
  check your findings against).
- **Wireshark Sample Captures wiki** — https://wiki.wireshark.org/SampleCaptures
  General-purpose protocol captures, useful for confirming the tool
  stays quiet on benign traffic.
- **Stratosphere IPS datasets** — https://www.stratosphereips.org/datasets-overview
  Labeled malware/botnet captures from a research lab at CTU University.

## Suggested demo flow

1. Download a port-scan or reconnaissance capture and run:
   ```
   python -m pcap_analyzer.cli path/to/capture.pcap --html report.html
   ```
2. Compare the findings against the write-up that accompanies the
   capture (most malware-traffic-analysis.net exercises include one) to
   sanity-check true/false positive rates.
3. Try a "clean" capture from the Wireshark wiki and confirm the tool
   reports no suspicious activity — this demonstrates the false-positive
   side of the story, which matters just as much as detection.

Keep downloaded captures out of the repo (the `.gitignore` already
excludes `*.pcap` outside of `tests/fixtures/`) — they're large, not
yours to redistribute, and not needed for anyone to run the tests.
